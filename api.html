static checkerMask(buffer)#

Applies a checkerboard mask to mask-out every other pixel in a buffer. This is used to reduce the amount of pixel information. This works because all important game entities are larger than a single pixel.

Parameters:

buffer (ndarray) – a (210,160) SECAM byte encoded image.

Returns:

a (210,160) SECAM byte encoded image with half the pixels masked to black.

Return type:

ndarray

static preprocess(buffer)#

Apply this method to reduce the dimensionality of the Atari’s screen from (210,160,3) to a (1344,) feature vector.

Parameters:

buffer (ndarray) – The Atari screen’s raw RGB data of shape (210,160,3).

Returns:

The Atari screen downsampled to a feature vector of shape (1344,)

Return type:

ndarray

static reduceDimensionality(buffer, dimensions=(42, 32))#

Downsamples a (210,160) image into a (42,32) image and returns a (1344,) feature vector. This operation is performed by splitting the image into ‘chunkSize’ non-overlapping chunks. Each non-overlapping chunk is a 5x5 image representing one pixel in the (42,32) image. The pixel in the (42,32) image is a colour uniformly sampled from the non-black colours within the chunk. The (42,32) image is flattened into a (1344,) feature vector.

Parameters:

  • buffer (ndarray) – a (210,160) SECAM byte-encoded image

  • dimensions (Tuple[int]) – the shape of the downsampled image. recommended: (42,32).

Returns:

Return type:

ndarray

getAction(teamPopulation, state, visited=[])#

Returns the team’s suggested action by choosing the action of the highest-bidding program. If the highest-bidding program is a reference to another team, recurse until the action is atomic. If the policy graph has a cycle, any team is only visited once.

Parameters:

  • teamPopulation (List[Team]) – All teams

  • state (array) – The feature vector representing the state/observation

  • visited (List[str]) – Used internally in recursive calls, ensures this method does not recurse forever when a cycle is encountered.

Returns:

an atomic action

Return type:

str

getFitness()#

This method is used to define the fitness of a team. By default, fitness is the score from the previous generation.

Returns:

The fitness score

id: str#

The unique identifier for the team

luckyBreaks: int#

This is a method of keeping elites. If a team is top-performing in a round, it is awarded a lucky break. In a future round, if has uncompetitive fitness and should be removed. The lucky break prevents its deletion during that generation. A lucky break is removed after use.

bid(state)#

Produces a bid which consists of an action and a confidence value of how certain the program is that the action is correct.

The bidding behaviour for a program is an emergent property.

The confidence value is simply the value stored in the program’s first register.

Mutator#

class mutator.Mutator#
static mutateInstruction(instruction)#

An instruction can be mutated by:

  • Changing its input source (program registers or state/observations from the environment).

  • Changing its operation

  • Changing the source register

  • Changing the destination register

It is guaranteed that a mutation will produce an instruction distinct from the original.

Parameters:

instruction (Instruction) – the instruction to mutate

Return type:

None

static mutateProgram(program)#

A program can be mutated by:

  • Deleting an instruction

  • Adding an instruction

  • Swapping an instruction

  • Mutating an instruction

An instruction will be deleted iff there is more than one instruction in the program.

An instruction will be added iff the instruction count would not exceed the maximum.

It is guaranteed that the mutated program will be distinct from original.

Parameters:

program (Program) – the program to mutate.

Return type:

None

static mutateTeam(programPopulation, teamPopulation, team)#

A team can be mutated by:

  • Adding an existing program

  • Deleting a program

  • Creating a new program and assigning it to the team

  • Changing a program’s action to a new action

  • Changing a program’s action to reference another team

An existing program will be added iff the program isn’t already used by the team.

A program will be deleted iff the program has at least one program remaining.

It is guaranteed that after mutation the team will have at least one atomic action.

Parameters:

  • programPopulation (List[Program]) – all programs at time of mutation

  • teamPopulation (List[Team]) – all teams at time of mutation

  • team (Team) – the team to mutate

class parameters.Parameters#

A class containing the model’s hyperparameters. Default values are provided for the Atari Frostbite environment.

ACTIONS: List = ['DO_NOTHING', 'STEER_LEFT', 'STEER_RIGHT', 'GAS', 'BRAKE']#

The list of possible actions for the environment.

ADD_INSTRUCTION_PROBABILITY: float = 0.7#

The probability of adding an instruction.

ADD_PROGRAM_PROBABILITY: float = 0.7#

The probability of adding a program to a team.

DELETE_INSTRUCTION_PROBABILITY: float = 0.7#

The probability of deleting an instruction.

DELETE_PROGRAM_PROBABILITY: float = 0.7#

The probabilty of deleting a program from a team.

ENVIRONMENT: str = 'CarRacing-v2'#

The name of the environment.

INITIAL_PROGRAM_POPULATION: int = 1000#

The number of programs generated in the first generation.

MAX_INITIAL_TEAM_SIZE: int = 5#

The maximum number of programs a team may have during creation.

MAX_INSTRUCTION_COUNT: int = 96#

The maximum number of instructions a program may have.

MUTATE_INSTRUCTION_PROBABILITY: float = 1.0#

The probability of mutating an instruction.

MUTATE_PROGRAM_PROBABILITY: float = 0.2#

The probability of mutating a program in a team.

NEW_PROGRAM_PROBABILITY: float = 0.2#

The probability of creating a new program and adding it to a team.

NUM_OBSERVATIONS: int = 27648#

The size of the state/observation space.

NUM_REGISTERS: int = 8#

The number of registers that all programs have.

POPGAP: float = 0.5#

The percentage of teams removed after a generation is completed.

POPULATION_SIZE: int = 360#

The quantity of root teams per generation.

On rampant mutation rounds, apply N rounds of mutation.
